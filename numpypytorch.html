NumPy

 

PyTorch is an open source machine learning library for developing deep learning models. PyTorch's popularity has been growing within the python community because its API integrates well with the rest of the python data science stack, making it easy to integrate into projects.

It also has a second less talked about use case though and that's for linear algebra / scientific computing, much like numpy.


Why use PyTorch?

The big advantage PyTorch offers is GPU support. PyTorch has GPU-accelerated versions of many of numpy's core functions, so can potentially speed up calculations on large arrays of data. The API is also like numpy's so it's easy to switch over to PyTorch without rewriting big chunks of code.


Why Stick With Numpy?

Numpy is one of the most used python libraries for scientific computing. The code is mature, and the API is stable, so you are not going to get any nasty surprises. Numpy is also a lot more lightweight than PyTorch, making it quicker and easier to test out new ideas with.

As you can see, the API for PyTorch is heavily influenced by Numpy and in most cases the functions have identical names. The most notable difference is that PyTorch refers to multi-dimensional arrays as tensors while Numpy calls them arrays.

Another difference is that PyTorch functions return a tensor object while Numpy returns ndarray. However, PyTorch has functions to convert between the two formats making it easy to move data from one library to the other.

Performance of PyTorch Vs Numpy

So, the API's are similar between PyTorch and Numpy but how does performance compare? Let's pick a few common functions and test them on a Google CoLab instance running a Tesla K80 GPU.
